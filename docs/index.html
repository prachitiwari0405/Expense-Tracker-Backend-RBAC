This API uses Role-Based Access Control (RBAC). Endpoints are protected by roles: ADMIN has full access, USER can manage only their own data. All protected routes require a valid JWT token in the Authorization header.
🔐 Authentication

All protected endpoints require a Bearer token obtained from the login endpoint.

HTTP Header
Authorization: Bearer <your_jwt_token>
Tokens expire after 24 hours. Re-authenticate to get a new token. Admin-only routes return 403 Forbidden if accessed with a USER role token.
🔑 Authentication Endpoints
POST /auth/register PUBLIC Register new user

Creates a new user account. Default role is USER. Passwords are stored as bcrypt hashes.

Request Body
application/json
{
  "name":     "Prachi Tiwari",
  "email":    "prachi@example.com",
  "password": "SecurePass@123"
}
Parameters
FieldTypeRequiredDescription
nameStringrequiredFull name of the user
emailStringrequiredUnique email address
passwordStringrequiredMinimum 8 characters
Responses
201User registered successfully. Returns user object without password.
400Validation error or email already exists.
POST /auth/login PUBLIC Authenticate & get JWT

Authenticates the user and returns a JWT token for subsequent requests.

Request Body
application/json
{
  "email":    "prachi@example.com",
  "password": "SecurePass@123"
}
Response 200 OK
application/json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "type":  "Bearer",
  "role":  "USER",
  "email": "prachi@example.com"
}
200Returns JWT token and user role.
401Invalid credentials.
💸 Expense Endpoints
POST /expenses USER Add new expense

Creates a new expense entry for the authenticated user.

Request Body
application/json
{
  "title":       "Grocery Shopping",
  "amount":      1250.50,
  "category":    "FOOD",
  "date":        "2025-04-05",
  "description": "Weekly groceries from BigBazaar"
}
Parameters
FieldTypeRequiredDescription
titleStringrequiredShort label for the expense
amountDoublerequiredExpense amount (positive)
categoryStringrequiredFOOD · TRANSPORT · HEALTH · SHOPPING · UTILITIES · ENTERTAINMENT · OTHER
dateLocalDaterequiredFormat: YYYY-MM-DD
descriptionStringoptionalAdditional notes
201Expense created. Returns expense object with generated ID.
400Validation failed.
401Unauthorized — missing or invalid token.
GET /expenses USER Get all my expenses

Returns all expenses for the currently authenticated user. Supports filtering and pagination.

Query Parameters
ParamTypeRequiredDescription
categoryStringoptionalFilter by category
startDateLocalDateoptionalFilter from date (YYYY-MM-DD)
endDateLocalDateoptionalFilter to date (YYYY-MM-DD)
pageIntegeroptionalPage number (default: 0)
sizeIntegeroptionalPage size (default: 10)
Response 200 OK
application/json
{
  "content": [
    {
      "id":          1,
      "title":       "Grocery Shopping",
      "amount":      1250.50,
      "category":    "FOOD",
      "date":        "2025-04-05",
      "description": "Weekly groceries"
    }
  ],
  "totalElements": 42,
  "totalPages":    5,
  "page":          0
}
GET /expenses/{id} USER Get expense by ID

Returns a single expense by its ID. Users can only access their own expenses.

200Returns expense object.
404Expense not found or doesn't belong to user.
PUT /expenses/{id} USER Update an expense

Updates an existing expense. All fields are optional — only provided fields will be updated.

application/json
{
  "amount":   1500.00,
  "category": "SHOPPING"
}
200Returns updated expense.
404Expense not found.
DELETE /expenses/{id} USER Delete an expense

Permanently deletes an expense by ID. Only the expense owner can delete it.

200Expense deleted successfully.
404Expense not found.

🛡️ Admin Endpoints
All admin endpoints require a JWT token with role ADMIN. Requests from USER role will receive 403 Forbidden.
GET /admin/users ADMIN Get all users

Returns a paginated list of all registered users in the system.

Response 200 OK
[
  { "id": 1, "name": "Prachi Tiwari", "email": "prachi@example.com", "role": "USER" },
  { "id": 2, "name": "Admin User",    "email": "admin@example.com",  "role": "ADMIN" }
]
GET /admin/expenses ADMIN Get all expenses

Returns all expenses from all users across the platform.

200Paginated list of all expenses with user info.
PUT /admin/users/{id}/role ADMIN Change user role

Promotes or demotes a user's role. Allows switching between USER and ADMIN.

application/json
{ "role": "ADMIN" }
200Role updated. Returns updated user object.
404User not found.
DELETE /admin/users/{id} ADMIN Delete a user

Permanently deletes a user and all their associated expense data.

200User and data deleted.
404User not found.
